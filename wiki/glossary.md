# Glossary

## Compounding Knowledge
*Canonical term.* Knowledge accumulated and cross-referenced such that each
new addition strengthens the whole. Contrast with stateless retrieval.
Origin: [[sources/karpathy-llm-wiki]]
See also: [[concepts/compounding-knowledge]]

## Convergence Architecture
*Canonical term.* Layered knowledge base system combining markdown wiki,
hybrid search, optional graph extraction, and MCP agent integration.
Origin: [[analyses/convergence-implementation]]

## Hybrid Search
*Canonical term.* Combining BM25 keyword search with vector semantic search
for fast, accurate retrieval over document collections.
See also: [[concepts/hybrid-search]]

## MCP (Model Context Protocol)
*Canonical term.* Open protocol for giving AI tools structured access to
external data and systems. Used for agent integration with Obsidian and
other knowledge bases.

## RAG
*Canonical term.* Retrieval-Augmented Generation. Stateless approach where
documents are chunked, embedded, and retrieved at query time.
Contrast: [[concepts/compounding-knowledge]]

## Stateful Agent
*Canonical term.* AI agent with persistent memory across sessions.
Examples: Letta, MemGPT-based systems, OpenClaw with long-term memory.
See also: [[concepts/agent-memory-types]]

## Wikilink
*Canonical term.* `\[\[Wikilink\]\]` syntax for bidirectional linking between
markdown pages. Used in Obsidian and this wiki for graph navigation.
