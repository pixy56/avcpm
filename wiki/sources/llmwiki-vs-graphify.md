# LLM Wiki vs Graphify: Comparing PKM Approaches

## Source

`raw/llmwiki-vs-graphify.md` — Written 2026-05-09 as a workspace research summary.

## Summary

This source compares two complementary approaches to AI-powered knowledge management that emerged in 2026: the **LLM Wiki** pattern (originated by Andrej Karpathy) and **Graphify** (by Safi Shamsi).

### Key Points

- **LLM Wiki** is a human-curated, LLM-maintained markdown wiki with `[[wikilinks]]`. Strengths: narrative-driven, compounding knowledge, Obsidian-compatible.
- **Graphify** is an auto-extraction tool that builds a queryable knowledge graph from any folder of files (code, docs, media). Strengths: machine-readable, fully automated, 71.5× token reduction.
- They are complementary: **Graphify is the auto-indexer; LLM Wiki is the auto-author.**
- Recommendation: use both together in an OpenClaw workspace.
